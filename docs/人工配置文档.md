> 本文档列出需要用户手动配置的内容，确保项目能顺利运行。

## 1. 环境要求

| 项目 | 版本 | 说明 |
|------|------|------|
| Node.js | ≥ v18.0 | 后端运行环境 |
| npm | ≥ v9.0 | 包管理器 |
| 浏览器 | Chrome/Firefox/Edge 最新版 | 前端运行 |
| Python 3 | ≥ v3.10 | 可选，用于生成角色立绘 |

**验证命令：**
```bash
node -v    # 应显示 v18.x.x 或更高
npm -v     # 应显示 9.x.x 或更高
```

## 2. 端口配置（如7100端口被占用）

默认端口为 **7100**。如需更改：

**步骤：**

1. 打开 `server.js`，找到第9行：
```javascript
const PORT = process.env.PORT || 7100;
```

2. 修改为其他端口，例如：
```javascript
const PORT = process.env.PORT || 8080;
```

3. 或使用环境变量启动：
```bash
PORT=8080 npm start
```

## 3. 内网穿透配置（分享给外网朋友）

如需让外网玩家访问你的本地服务器：

### 方案A：使用 ngrok（推荐）

1. 注册 [ngrok](https://ngrok.com) 并下载客户端
2. 获取 authtoken 并完成初始化
3. 启动游戏服务器后，在另一个终端运行：
```bash
ngrok http 7100
```
4. 复制生成的 `https://xxxx.ngrok-free.app` 链接发给朋友
5. 朋友打开该链接即可直接游戏（WebSocket会自动连接）

### 方案B：使用 frp

1. 配置 frp 服务端和客户端
2. 将本地 7100 端口映射到公网服务器

### 方案C：使用 cpolar / 花生壳

类似 ngrok 的配置流程。

**注意：** 内网穿透URL每次重启可能会变化，需要重新分享。

## 4. 角色立绘重新生成

如需重新生成或自定义角色立绘：

**步骤：**

1. 确保 Python 3 和 Pillow 已安装：
```bash
python3 -c "from PIL import Image; print('OK')"
```

2. 编辑 `generate_characters.py` 文件中的颜色配置：
```python
characters = [
    {
        "primary": (220, 60, 60, 255),      # 主体颜色 (R,G,B,A)
        "secondary": (255, 120, 80, 255),   # 高光颜色
        "hair": (255, 100, 50, 255),         # 头发颜色
        "name": "火焰小子",
        "path": os.path.join(BASE_DIR, "public/assets/characters/red_boy.png")
    },
    # ... 其他角色
]
```

3. 运行生成脚本：
```bash
python3 generate_characters.py
```

4. 生成的图片会保存到 `public/assets/characters/` 目录

## 5. 添加自定义地图

如需添加自定义地图：

**步骤：**

1. 打开 `src/backend/mapGenerator.js`
2. 在 `MAP_PRESETS` 中添加新地图配置：
```javascript
const MAP_PRESETS = {
  // ... 现有地图
  mymap: {
    name: '我的地图',
    description: '自定义地图描述',
    layout: null  // 使用随机生成，或手动定义layout
  }
};
```

3. 添加自定义生成函数（可选）：
```javascript
function generateMyMap(map) {
  // 手动填充 map[y][x] 的瓦片值
  // 0=空地, 1=石头, 2=砖块
  for (let y = 1; y < MAP_HEIGHT - 1; y++) {
    for (let x = 1; x < MAP_WIDTH - 1; x++) {
      map[y][x] = /* 你的逻辑 */;
    }
  }
}
```

4. 在 `generateMap` 函数中添加分支：
```javascript
switch (presetName) {
  // ...
  case 'mymap': generateMyMap(map); break;
}
```

5. 在 `public/index.html` 的地图选择下拉框中添加新选项：
```html
<option value="mymap">我的地图</option>
```

## 6. 修改游戏参数

如需调整游戏平衡性：

**打开 `src/backend/gameLoop.js`，修改以下常量：**

| 常量 | 默认值 | 说明 |
|------|--------|------|
| `TICK_RATE` | 20 | 游戏循环频率（次/秒） |
| `BOMB_TIMER` | 3.0 | 炸弹爆炸时间（秒） |
| `ITEM_DROP_CHANCE` | 0.4 | 砖块掉道具概率（40%） |
| `ITEM_WEIGHTS` | [0.3,0.3,0.25,0.1,0.05] | 各道具权重 |

**打开 `src/backend/roomManager.js`，修改游戏时长：**

```javascript
this.gameDuration = 180; // 3分钟（秒）
```

## 9. 无网络环境运行（离线模式）

如需在无网络环境运行：

**问题：** Phaser.js 通过 CDN 加载，无网络时无法加载。

**解决方案：**

1. 下载 Phaser.js 到本地：
```bash
curl -o public/libs/phaser.min.js https://cdn.jsdelivr.net/npm/phaser@3.60.0/dist/phaser.min.js
```

2. 修改 `public/index.html` 中 Phaser 的引用：
```html
<!-- 原引用 -->
<script src="https://cdn.jsdelivr.net/npm/phaser@3.60.0/dist/phaser.min.js"></script>

<!-- 改为本地 -->
<script src="libs/phaser.min.js"></script>
```

## 10. 常见问题

### Q: 启动后浏览器无法访问？

检查：
1. 防火墙是否放行了 7100 端口
2. 是否已有其他程序占用了 7100 端口
3. 访问地址是否正确（`http://localhost:7100`）

### Q: 外网玩家无法连接？

需要配置内网穿透（见第3节）。本地服务器默认只在本地网络可访问。

### Q: 游戏运行一段时间后变得很慢？

可能是：
1. 同时运行太多房间（建议限制10个以内）
2. 内存泄漏（Node.js 长时间运行后可能需要重启）

### Q: 如何重置所有房间？

重启服务器即可，所有房间数据存储在内存中。
