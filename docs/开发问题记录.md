# AI Q版泡泡堂 · 开发过程问题记录

> 记录开发过程中遇到的 Bug、根因分析、修复方案及经验教训。

---

## 一、阻断级 Bug

### 1. `gameLoop.js` 重复代码 — 服务器无法启动

**现象**：执行 `node server.js` 即报错退出，`SyntaxError: Unexpected token '.'`

**根因**：`spawnItem()` 方法体被完全重复。方法在 342 行正常结束后，343-363 行出现了一份完全相同的代码，挂在类体中间（不在任何方法内），Node.js 解析时报语法错误。

```javascript
// 正确的方法结束
  spawnItem(x, y) {
    // ... 正常逻辑
    this.room.items.push({ x, y, type });  // line 342
  }
    if (Math.random() > ITEM_DROP_CHANCE) return;  // ❌ line 343 重复！
    // ... 一整段重复代码
```

**修复**：删除 343-363 行重复代码块。

**教训**：
- 代码合并/复制粘贴后必须检查上下文边界
- 启动测试应该是 CI 的第一步

---

### 2. 炸弹动画后纹理帧越界 — 24 条控制台警告 + 8 异常

**现象**：放置炸弹后，控制台持续输出 `Texture "char_bomb" has no frame "12"`，伴随 8 个空 JS 异常。

**根因**：炸弹动画播放后，Phaser sprite 的纹理从 `char_walk`（16 帧）切换到了 `char_bomb`（仅 4 帧）。动画结束后 `_placing` 变为 false，行走代码执行 `sprite.setFrame(12)`，但 `char_bomb` 纹理只有 0-3 帧。

```javascript
// 炸弹动画 → 纹理变为 char_bomb（4帧）
sprite.play('w0-bomb');

// 动画结束 → 纹理仍是 char_bomb → 设帧 12 失败！
sprite.anims.stop();
sprite.setFrame(12);  // ❌ char_bomb 无帧 12
```

**修复**：在 `.setFrame()` 前添加纹理还原：
```javascript
if (sprite.texture.key !== C.walk) {
    sprite.setTexture(C.walk);  // 还原为行走纹理
}
sprite.setFrame(frameIdx);
```

**教训**：
- Phaser 动画会改变 sprite 的 texture 引用
- `.anims.stop()` 不会还原纹理
- 任何跨纹理的帧操作都需要显式 setTexture

---

### 3. 随机石头堵死出生点

**现象**：部分地图生成后，四个角落出生点之间被石头完全隔断，角色被堵在角落无法移动。

**根因**：`addRandomStones()` 纯随机放置石头，没有任何连通性验证。180° 旋转对称只保证公平，不保证可达。

**修复**：
1. 石头放置后运行 BFS 遍历，从 (1,1) 检查能否到达其他三个出生点
2. 不通过则减少石头数量重试（最多 10 次）
3. 最终兜底：清除所有内部石头，仅保留边缘

```javascript
// 验证连通性
const spawns = [[1,1], [11,1], [1,9], [11,9]];
if (spawns.every(sp => bfsReachable(map, spawns[0][0], spawns[0][1], sp[0], sp[1]))) return;
count = Math.max(count - 2, 4);  // 减石重试
```

**验证**：200/200 地图（4 种类型 × 50 次）全部连通。

## 二、架构/逻辑 Bug

### 4. 非默认角色皮肤闪退

**现象**：白团子角色正常，企鹅/恐龙/小鸡皮肤进入游戏后崩溃退回大厅。

**根因**：角色精灵图的动画创建代码（`generateFrameNumbers` + `anims.create`）没有 try-catch 保护。如果某角色的精灵图加载异常或帧数不匹配，整个 `createCharAnimations()` 抛出异常，导致 Phaser 场景创建失败。

**修复**：三层 try-catch 加固
1. **动画创建层**：整个 `generateFrameNumbers` + `anims.create` 包 try-catch，失败则 `c.ok = false` 回退程序化纹理
2. **行走动画层**：`setFrame`/`play`/`setTexture` 包 try-catch，`console.warn` 降级
3. **玩家渲染层**：整个 `updatePlayers` 循环包 try-catch，单个玩家异常不拖垮全场

---

### 5. 房间 ID 复用撞车

**现象**：玩家退出后重进，新玩家被旧连接的断线事件误删。

**根因**：玩家 ID 按 `players.size + 1` 生成，玩家退出后大小减少，新玩家可能复用旧 ID。旧 WebSocket 断线时 `handleDisconnect` 拿着旧 ID 删了新玩家。

**修复**（早期迭代）：
- 玩家/Bot ID 改用单调递增计数器 `nextPlayerNum`
- 同名重复加入时 `removePlayerByName()` 踢掉旧身份
- 最后一人离开即销毁房间

## 三、体验问题

### 6. 浏览器缓存导致修改不生效

**现象**：修改 `game.js` 后浏览器不加载新版本，仍报旧错误。

**根因**：浏览器强缓存 JS 文件，`<script src="js/game.js">` 无版本号。

**修复**：HTML 中 JS 引用加版本号：`<script src="js/game.js?v=b6">`

**教训**：开发阶段 JS/CSS 引用必须带缓存破坏参数。

---

### 7. Phaser CDN 偶发不可用

**现象**：jsdelivr CDN 波动导致全站不可用。

**修复**：`phaser.min.js`（1.1MB）下载到 `public/js/lib/` 本地托管。

**教训**：本地项目应零外部依赖。

---

### 8. 端口占用 / 进程残留

**现象**：反复重启服务器时端口 3000 被占。

**根因**：`&` 后台启动的进程 + `Ctrl+C` 未完全清理。

**应对**：
- 启动前 `netstat | grep :3000 | awk ... taskkill`
- 固定端口 3000，移除 `process.env.PORT` 动态读取

---

## 四、经验总结

| # | 原则 |
|---|------|
| 1 | **服务器必须先跑通**：一个 class-level 语法错误就让整个项目瘫痪 |
| 2 | **动画纹理要还原**：Phaser `play(anim)` 会改纹理，`stop()` 不会还原 |
| 3 | **随机生成要验证**：地图石头必须有 BFS 连通性检查 |
| 4 | **顶层代码要防爆**：单点异常不应崩溃整个场景 |
| 5 | **本地项目零外部依赖**：CDN 是单点故障，本地化成本极低 |
| 6 | **ID 生成用单调递增**：`size+1` 在增删场景下会复用撞车 |
| 7 | **开发阶段必加缓存破坏**：`?v=xxx` 省去无数次误判 |
