# AI Q版泡泡堂 · 完整方案文档

> 版本：v1.0（Grilling 终版）
> 目标：浏览器端Q版泡泡堂，支持真人联机 + 内置AI人机 + 外部Agent直接接入
> 预算：零成本/白嫖

---

## 1. 项目概述

做一个能在浏览器直接打开的Q版泡泡堂对战游戏。核心特色：
- **玩家 vs 玩家**：创建房间，分享链接，真人实时对战
- **玩家 vs 内置人机**：开房间时可选难度，空位自动补Bot
- **玩家/Agent vs 外部Agent**：分享现成Agent脚本，对方复制粘贴即可运行参战
- **观战系统**：实时上帝视角观战，支持赛后战绩回放

美术走AI生成Q版角色立绘，程序用简单动画模拟（后续可升级骨骼动画）。

## 3. 前端方案：Phaser.js 安装与使用

### 3.1 安装 Phaser.js（三种方式）

#### 方式一：CDN引入（最简单，推荐MVP阶段）

在HTML `<head>` 中加入：

```html
<script src="https://cdn.jsdelivr.net/npm/phaser@3.60.0/dist/phaser.min.js"></script>
```

不需要下载任何文件，浏览器自动加载。

#### 方式二：npm安装（如果你用Node.js构建工具）

```bash
# 创建项目目录
mkdir q-paopaotang && cd q-paopaotang

# 初始化npm项目
npm init -y

# 安装Phaser.js
npm install phaser
```

安装后，在JS文件中引入：

```javascript
import Phaser from 'phaser';
```

#### 方式三：直接下载文件（离线/内网环境）

1. 打开 Phaser GitHub Releases：https://github.com/phaserjs/phaser/releases
2. 下载 `phaser.min.js`（或 `phaser.js` 开发版）
3. 放到你的项目 `libs/` 目录下
4. HTML中引入：

```html
<script src="libs/phaser.min.js"></script>
```

### 3.2 Phaser.js 核心概念（泡泡堂需要用的）

| Phaser概念 | 泡泡堂里的对应 | 说明 |
|-----------|-------------|------|
| `Scene` | 游戏场景 | 菜单Scene、房间Scene、对战Scene、结算Scene |
| `Tilemap` | 地图网格 | 13×11的瓦片地图，石头/砖块/空地三种瓦片 |
| `Sprite` | 角色、炸弹、道具 | 角色走上下左右4方向，炸弹有倒计时动画 |
| `Group` | 炸弹组、道具组、爆炸组 | 方便批量更新和碰撞检测 |
| `Timer` | 炸弹倒计时、局内倒计时 | 3分钟局时、炸弹3秒爆炸 |
| `Particle` | 爆炸特效、道具拾取特效 | 砖块炸碎粒子、爆炸火焰扩散 |

## 4. 后端方案：房间 + 状态同步 + Agent接入

### 4.1 后端核心职责

1. **房间管理**：创建、加入、离开、销毁房间
2. **状态权威**：地图状态、玩家位置、炸弹倒计时、道具生成、胜负判定
3. **消息广播**：WebSocket向房间内所有客户端广播状态更新
4. **Agent接口**：HTTP API供外部Agent查询状态、提交动作
5. **战绩持久化**：每局结束写入Upstash Redis

### 4.2 房间系统

```javascript
// 房间数据结构示例
const room = {
  id: "BV8C",           // 4位随机字母数字房间号
  mode: "1v1",          // 1v1 | ffa(混战) | 2v2 | pve(人机)
  phase: "waiting",     // waiting | playing | ended
  map: [[...]],         // 13×11瓦片地图（后端生成权威地图）
  players: {
    "p1": { x:1, y:1, bombs:1, power:1, speed:1, lives:2, team:"red" },
    "p2": { x:11, y:9, bombs:1, power:1, speed:1, lives:2, team:"blue" }
  },
  bots: [],             // 内置AI人机配置
  agents: [],           // 外部Agent连接
  spectators: [],       // 观战者
  startTime: null,      // 开局时间戳
  gameLoop: null        // 后端游戏循环引用
};
```

### 4.3 状态同步策略

| 数据类型 | 同步方式 | 频率 | 说明 |
|---------|---------|------|------|
| 玩家位置 | 客户端预测 + 服务器校正 | 玩家操作时立即发，服务器广播 | 减少延迟感 |
| 炸弹放置 | 客户端请求→服务器确认→广播 | 即时 | 防止作弊 |
| 爆炸/砖块破坏 | 服务器权威计算→广播 | 炸弹计时到时 | 客户端只播放动画 |
| 道具生成 | 服务器权威→广播 | 砖块破坏时 | 客户端同步显示 |
| 胜负判定 | 服务器权威→广播 | 命数归零或超时 | 客户端显示结算画面 |
| 局内倒计时 | 服务器心跳→广播 | 每秒1次 | 同步所有客户端时间 |

### 4.4 后端游戏循环（关键）

后端需要维护一个**确定性游戏循环**（固定tick，如每秒20次）：

```javascript
// 伪代码
setInterval(() => {
  // 1. 处理所有待处理动作（移动、放炸弹、踢炸弹）
  // 2. 更新炸弹倒计时，到0则触发爆炸
  // 3. 爆炸扩散计算（火焰十字扩散，遇到石头停止，砖块破坏并生成道具）
  // 4. 碰撞检测（火焰与玩家、道具与玩家）
  // 5. 判定死亡/掉命/道具拾取
  // 6. 检查胜负条件（一方命数归零、超时）
  // 7. 更新Agent可见状态（上帝视角完整地图）
  // 8. 广播状态给所有客户端和Agent
}, 1000 / 20); // 20 tick/s
```

**为什么后端要有游戏循环？** 防止客户端作弊，确保Agent和玩家看到的都是权威状态。

## 5. 游戏机制设计

### 5.1 地图系统

- **尺寸**：13列 × 11行（经典泡泡堂比例）
- **瓦片类型**：
  - `0` = 空地（可行走、可放炸弹）
  - `1` = 固定石头（不可破坏、不可行走、炸弹火焰被阻挡）
  - `2` = 可破坏砖块（炸毁后变空地，概率掉落道具）
- **生成规则**：
  - 四角出生点周围2×2区域固定为空地（防止开局被堵）
  - 地图边缘一圈固定石头
  - 内部随机分布石头和砖块（固定比例，如石头30%、砖块40%、空地30%）
  - 提供**3-5张经典地图**，开房间时随机选或房主选

### 5.2 道具系统

| 道具 | 效果 | 图标风格 | 叠加上限 |
|------|------|---------|---------|
| 炸弹 | 炸弹数量+1 | 红色炸弹图标 | 最多8个 |
| 火力 | 炸弹爆炸范围+1格 | 红色火焰图标 | 最多8格 |
| 速度 | 移动速度+1档 | 蓝色鞋子图标 | 最多5档 |
| 踢炸弹 | 可踢动前方炸弹 | 绿色足球图标 | 有/无 |
| 穿墙 | 可穿过砖块（非石头） | 紫色幽灵图标 | 有/无（限时） |

道具从被炸毁的砖块中**概率掉落**（如40%掉落率，随机选一种）。

### 5.3 胜负规则（按模式）

| 模式 | 人数 | 胜利条件 | 超时结算（3分钟） |
|------|------|---------|----------------|
| **1v1对战** | 2人 | 一方命数归零 | 命数多者赢，相同平局 |
| **自由混战(FFA)** | 2-4人 | 最后存活者赢 | 击杀数多者赢，相同平局 |
| **2v2组队** | 4人 | 敌方全队命数归零 | 队伍剩余命数总和多者赢 |
| **人机挑战(PVE)** | 1人+3Bot | 玩家存活到最后 | 击杀Bot数 |

- **单局2命**：初始2命，被炸到-1命，命数归零出局
- **无敌时间**：复活/开局后3秒无敌，闪烁提示
- **平局处理**：显示"平局"，双方不计胜负

### 5.4 角色系统（AI生成Q版）

- **角色数量**：MVP阶段4-6个可选角色
- **生成方式**：用AI生成正面/背面/左面/右面4方向立绘 + 胜利/失败表情
- **动画方式**：Phaser中简单缩放/弹跳模拟行走（用1张图+程序变形），后续升级为精灵图/骨骼动画
- **角色差异**：只有外观差异，无属性差异（公平竞技）

## 6. Agent系统（双轨设计）

### 6.1 轨道一：内置人机（Bot）

- **开房间时可选**：房主选难度，空位自动填入Bot
- **难度等级**：
  - **简单**：直线走，随机放炸弹，不捡道具，不躲爆炸
  - **中等**：会躲自己炸弹，会捡道具，不主动追击
  - **困难**：会算安全区域，会封路，会追击，优先抢火力/炸弹
  - **疯子**：贴脸追杀，高风险高回报，会利用踢炸弹阴人
- **Bot行为**：直接跑在后端游戏循环里，作为"虚拟玩家"参与状态同步

### 6.2 轨道二：外部Agent（核心特色）

**设计理念**：不是让外部AI"读文档→写代码→调试"，而是给**现成可运行的脚本**，对方复制粘贴、填个名字就跑。

#### 分享内容（创建房间后生成）

玩家点击"邀请Agent"后，系统生成一段分享文本：

```markdown
【Q版泡泡堂 · Agent对战邀请】

房间号：BV8C
服务器：ws://your-server.com
观战链接：http://your-server.com/?room=BV8C&spectate=1

👉 把下面这段代码给你的AI Agent（ChatGPT/Codex/Claude），让它直接运行参战：

--- 复制下面整段 ---

你是Q版泡泡堂对战Agent，现在直接加入对战。
服务器地址：ws://your-server.com
房间号：BV8C

操作方式：
1. 连接WebSocket：ws://your-server.com/agent?room=BV8C
2. 发送加入消息：{"type":"join","name":"你的名字","kind":"agent"}
3. 等待服务器返回 playerId 和 token
4. 等待 phase=playing 开始作战
5. 游戏中循环：
   - 接收服务器推送的 state（完整地图状态，上帝视角）
   - 分析：玩家位置、炸弹位置及倒计时、道具位置、安全区域
   - 发送动作：{"type":"action","move":"up/down/left/right","placeBomb":true/false}
6. 局结束后接收 result 看排名

规则提醒：
- 单局2命，被炸-1命，命数归零出局
- 炸弹3秒后爆炸，火焰十字扩散，石头挡火
- 炸砖块有概率掉道具（炸弹/火力/速度/踢炸弹/穿墙）
- 3分钟超时，比命数/击杀数
- 开局3秒无敌

--- 复制结束 ---

⚡ 你的Agent只需要运行这段逻辑，不需要自己写接入代码！
```

#### 外部Agent接入协议（后端实现）

```javascript
// Agent WebSocket连接示例
ws://your-server.com/agent?room=BV8C

// Agent → 服务器
{"type":"join", "name":"ClaudeAgent", "kind":"agent"}
{"type":"action", "move":"right", "placeBomb":true, "kick":"up"}

// 服务器 → Agent（状态推送）
{
  "type":"state",
  "phase":"playing",
  "tick": 1523,
  "map": [[0,1,2,...], ...],  // 13×11完整地图
  "players": {
    "p1": {"x":3, "y":5, "bombs":2, "power":3, "lives":2, "invincible":false},
    "p2": {"x":8, "y":2, "bombs":1, "power":2, "lives":1, "invincible":true}
  },
  "bombs": [
    {"x":4, "y":5, "owner":"p1", "timer":1.2, "power":3}
  ],
  "items": [
    {"x":6, "y":4, "type":"power"}
  ],
  "timeLeft": 145
}
```

**关键**：Agent接收的是**完整结构化状态（JSON）**，不是像素画面，所以任何LLM/代码Agent都能直接解析并决策。
